# Contributing

Thank you for your interest in contributing to this project.

## Workflow
- Fork the repository and create a feature branch
- Make your changes in that branch
- Open a pull request describing the change and why it is needed

## Code style
- Follow the existing coding style and structure
- Run the formatter (Black) before submitting
- Keep functions and modules small and focused

## Tests
- All new code must include accompanying tests
- Existing tests must continue to pass
- Use pytest, and pytest-qt for GUI-related tests

## Discussions
If you’re planning a larger change, please open an issue first so we can discuss the approach.
